## Visual Studio Computer Use

### Build and Run

Prefer menu or toolbar controls with accessible labels for Build, Start, Stop, and Test Explorer actions. Avoid coordinate clicks in dense toolbars unless the target is confirmed by `get_app_state`.

### Editing

If a file is open in the editor, focus the editor before typing. For code changes larger than a few lines, prefer filesystem edits through Codex tools over GUI typing.

### Dialogs

Visual Studio shows modal dialogs for package restore, trust prompts, migrations, and debug target selection. Resolve visible dialogs before attempting more editor actions.

### Localization

Prefer command IDs, AutomationId, control type, supported UI Automation patterns, and keyboard shortcuts over English visible labels.
